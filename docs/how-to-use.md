# How to Use These Resources

These materials are designed to support your learning, not replace the effort of working through problems yourself. You will learn far more by attempting each exercise before consulting the suggested solution.

## Two Kinds of Problems

Each chapter contains two categories of end-of-chapter material, and this site treats them differently.

**Review Questions** test conceptual understanding. For these, you will find a concise written answer that captures the key points. Use them to check your understanding after attempting the question on your own.

**Hands-On Exercises** ask you to build something. For these, the [Selected Solutions](solutions/index.md) give an outline or a short reference where appropriate. The larger project exercises are left for independent practice, since they have no single fixed answer. Where a reference implementation exists, you will find it in the [Source Code](code/index.md) section.

## Recommended Workflow

1. Read the chapter and attempt the questions and exercises before opening any solution.
2. For conceptual questions, write your own answer first, then compare.
3. For coding exercises, build your own version, get it working, then study the reference implementation to compare approaches.
4. Type the example code yourself rather than copying and pasting. The physical act of typing, debugging, and modifying code commits knowledge to memory.

!!! warning "Academic integrity"
    If you are using this book in a course, follow your instructor's policy on the use of published solutions. These materials are intended to support learning, not to be submitted as your own work.

## Running the Code

Most front-end examples (HTML, CSS, JavaScript) run directly in any modern web browser, so you just open the `.html` file. Server-side examples (PHP, SQL/MySQL) require a local server environment such as XAMPP, MAMP, or Docker. Each `src/` subfolder includes a short `README` describing how to run that example.
